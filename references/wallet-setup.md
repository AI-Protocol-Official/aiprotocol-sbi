# Wallet Setup Reference

> **When to use this reference:** Use this file when you need detailed information about creating, connecting, or checking wallet status. For the overall workflow, see [SKILL.md](../SKILL.md).

All wallets are **local encrypted keystores** generated by ethers.js, stored at `~/.aiprotocol-sbi/keystores/`. The private key never leaves the user's machine.

---

## 1. Create Wallet

Generate a new encrypted wallet for an agent.

### Command

```bash
aiprotocol-sbi wallet create --bot-name "<NAME>" --json
```

### Parameters

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--bot-name <name>` | string | `"Agent"` | Agent name. Derives a stable bot ID via SHA-256(`name + network`). |
| `--bot-id <id>` | string | _(derived)_ | Explicit UUID. Overrides name-derived ID. |
| `--force` | boolean | `false` | Create new wallet even if one exists for this bot. |
| `--json` | boolean | `false` | Machine-readable output. |

### Response — New Wallet Created

```json
{
  "ok": true,
  "data": {
    "action": "created",
    "botId": "7e344b4f-7cc0-84aa-9e8b-242bab4bda14",
    "botName": "TestAgent",
    "address": "0x6136c91b69e271bbF40DFC1a44bd52A17B8db94E",
    "keystorePath": "~/.aiprotocol-sbi/keystores/7e344b4f-7cc0-84aa-9e8b-242bab4bda14.json",
    "provider": "ethers.js (local encrypted keystore)",
    "createdAt": "2026-02-17T11:01:37.174Z"
  }
}
```

### Response — Existing Wallet Reused

If the same bot name/ID already has a keystore, the existing wallet is returned:

```json
{
  "ok": true,
  "data": {
    "action": "reused",
    "botId": "7e344b4f-7cc0-84aa-9e8b-242bab4bda14",
    "botName": "TestAgent",
    "address": "0x6136c91b69e271bbF40DFC1a44bd52A17B8db94E",
    "provider": "ethers.js (local encrypted keystore)"
  }
}
```

Use `--force` to create a new wallet even if one exists.

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `action` | string | `"created"` or `"reused"` |
| `botId` | string | Deterministic UUID for this bot |
| `botName` | string | Human-readable agent name |
| `address` | string | Ethereum wallet address (EIP-55 checksummed) |
| `keystorePath` | string | Path to encrypted keystore file |
| `provider` | string | Always `"ethers.js (local encrypted keystore)"` |
| `createdAt` | string | ISO 8601 timestamp (only on `"created"`) |

## 2. Connect Existing Keystore

Connect a previously created local keystore by bot ID.

### Command

```bash
aiprotocol-sbi wallet connect --bot-id "<BOT_ID>" --json
```

Or interactively pick from available keystores:

```bash
aiprotocol-sbi wallet connect --list
```

### Parameters

| Flag | Type | Default | Description |
|------|------|---------|-------------|
| `--bot-id <id>` | string | _(prompted)_ | Bot ID of the keystore to connect. |
| `--list` | boolean | `false` | List all local keystores and pick interactively. |
| `--json` | boolean | `false` | Machine-readable output. |

### Response

```json
{
  "ok": true,
  "data": {
    "address": "0xABcD...1234",
    "botId": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
    "chainType": "ethereum",
    "balance": {
      "ali": "2500.00",
      "native": "0.05"
    },
    "provider": "ethers.js (local encrypted keystore)"
  }
}
```

## 3. Check Identity (`wallet who`)

Check whether this machine has a registered bot identity and keystore.

### Command

```bash
aiprotocol-sbi wallet who --json
```

### Response — Bot Found

```json
{
  "ok": true,
  "data": {
    "registered": true,
    "keystoreFound": true,
    "botId": "7e344b4f-7cc0-84aa-9e8b-242bab4bda14",
    "botName": "TestAgent",
    "address": "0x6136c91b69e271bbF40DFC1a44bd52A17B8db94E",
    "keystorePath": "~/.aiprotocol-sbi/keystores/7e344b4f-7cc0-84aa-9e8b-242bab4bda14.json",
    "provider": "ethers.js (local encrypted keystore)",
    "firstSeen": "2026-02-17T11:01:36.901Z",
    "lastSeen": "2026-02-17T11:01:36.901Z"
  }
}
```

### Response — No Bot Registered

```json
{
  "ok": true,
  "data": {
    "registered": false,
    "message": "No bot identity found. Run `aiprotocol-sbi wallet create` to register."
  }
}
```

## 4. Check Balance & Eligibility (`wallet status`)

Check ALI and ETH balance, and whether the wallet can launch an economy.

### Command

```bash
aiprotocol-sbi wallet status --json
```

Or check a specific address:

```bash
aiprotocol-sbi wallet status --address 0xABcD...1234 --json
```

### Response

```json
{
  "ok": true,
  "data": {
    "address": "0x6136c91b69e271bbF40DFC1a44bd52A17B8db94E",
    "network": "base",
    "balance": {
      "ali": "2500.00",
      "native": "0.05"
    },
    "isEligibleForLaunch": true,
    "requiredAli": "1000.00",
    "shortfallAli": "0",
    "lastCheckedAt": "2026-02-17T11:01:50.088Z"
  }
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `address` | string | Wallet address |
| `network` | string | `"base"` |
| `balance.ali` | string | ALI token balance |
| `balance.native` | string | ETH balance (for gas) |
| `isEligibleForLaunch` | boolean | `true` if wallet has enough ALI |
| `requiredAli` | string | Minimum ALI needed to launch |
| `shortfallAli` | string | How much more ALI is needed (`"0"` if eligible) |

## Keystore Architecture

```
~/.aiprotocol-sbi/
├── config.json                                    # CLI config (never commit)
└── keystores/
    └── <botId>.json                               # EIP-55 encrypted keystore (ethers.js)
```

- **Format**: Standard Web3 Secret Storage (compatible with MetaMask, Geth, Hardhat)
- **Encryption**: AES-128-CTR with scrypt key derivation, password set during `setup`
- **One keystore per bot**: Bot ID is deterministic from `SHA-256(botName + network)`
- **Backup**: The `keystores/` directory is the only copy of the private key. Back it up.

## Error Cases

| Error | Cause |
|-------|-------|
| `{"ok": false, "error": "Config not found"}` | Run `aiprotocol-sbi setup` first |
| `{"ok": false, "error": "No bot identity found"}` | Run `wallet create` first |
| `{"ok": false, "error": "Keystore not found"}` | Keystore file missing or deleted |
| `{"ok": false, "error": "Invalid password"}` | Wrong keystore password |
